1 Protozoa

Purpose Built Single Celled Organism for Educational Content.

Protozoa makes delivering on-line content faster & easier because it was designed, from the beginning, to be used in an educational setting. It’s made for educators of varying skill levels to produce attractive, accessible, responsive learning content for your students. A question I get a lot is, why didn’t you just use Bootstrap. We did take a hard look at Twitter Bootstrap, and found out very quickly that it was too open a tool. In it’s own words it was designed to be used on “Projects of all sizes” and it was “Made for folks of all skill levels.” This definition was just too broad for our needs.

An educator's online content usually lives in some sort of learning management system like BlackBoard or Canvas, but that is only the technical story. There is usually a whole information hierarchy that exists above that your content fits into. The content object itself exists within lesson or modules. Those are usually contained within courses which are in turn contained within larger programs or schools. Protozoa takes all that taxonomy and translates it for you into a pre-existing semantic html structure that you can use as is, or change to fit your needs.

There are so many different roles in education pointed directly at creating content for online courseware, but most have little to do with front end web design. Some professional examples include people who are instructors, instructional designers, subject matter experts, professional development coordinators, and IT folk. All of whom may have experience using the web, but may have little to no training in the intricacies involved with front end web design, or 508/504 compliance, much less the need for accessible feature sets. Protozoa takes care of most of that for you.

The Demo

Many times all these words mean nothing until you can see it in action. With that in mind we have created not only this very website from Protozoa, but took and inserted some example content, and example widgets in our interactive demo.

A Truly Accessible Shell

We work at a State college, and as such are required to follow the best practices outlined for section 504 & 508 of the Rehabilitation Act, and the Americans with Disabilities Act (ADA). We considered this the low bar that we had to hit, and took extra pains to curate a positive experience for people who require assistive technology to access our content.

We believe no one should have a bad experience consuming our content just because they need to take an online course, no matter the circumstances.

True Device Agnosticism

We have all seen it. There is a new device coming out every day, and many of these devices can access the web. Things like “smart” refrigerators are on the market. It’s just too much for any one person to keep track of. The good news is you’re not expected to, and we can still make it work using a web design technique called responsive design. To see it in action, just take your browser right now and change the width of the browser window until it enters the mobile width realm. All your content just flows into place like it was meant to be on that mobile phone in the first place.

2 Getting Started

Download & Install

  • Download the latest release from the Github repository then Click on the "source code" link to initiate the download.

  • Install by un-compressing the archive and using the resulting folder to build a single learning object.

File Structure

  • /css (folder)

  • /img (folder)

  • /js (folder)

  • index.html

  • license.html

  • template.html

Protozoa, just like the name implies, is meant to be a single celled organism—The smallest particle of educational content. That being said, we designed it to be portable from the beginning. We meant for this to be simple from the start so the file structure is as straight forward as we could make it.

The "css" Folder

We did use SASS to make editing styles quicker. What this means is that for each css file you will find it’s accompanying SCSS file. If you know SASS and would like to take advantage of its features, and you have to tools to do so, go ahead and edit then compile the SCSS files. Just make sure that your SCSS output still matched the file listed within the head of the index.html file.

If you prefer not to bother with learning SASS then go ahead and just delete those and work directly with the CSS files. To customize your LO edit the “styles.css” file.

The "img" Folder

This folder holds any images you may have. If you keep them in this folder they will naturally travel with your content. You will find some images already existing here that are used in the interface itself.

It is your choice whether you would like to change these out for ones that match your color brand closely, but we choose to keep them the way they are because they were chosen to be neutral in nature to work with any colored interface.

The index.html File

This is an example file, with example content inserted. You can use it as a working example of how to insert content within each slide. First open this file in a web browser, then open the same file in a txt editor. That way you can see the result of code rendered to the screen.

The template.html File

This file contains a set of thirty blank slides. When starting a new learning object start here by filling out as many slides as you need, then deleting the rest. As a team we have found that it’s easier and faster to delete slides than it is to have one that you copy paste as you go along. The reason I put 30 slides in there is because after 4 years of doing this, no one has ever asked for more than that—ever—in 4 years.

The license.txt File

This file must be included with each learning object you make, because it outlines the fact that this is under the MIT license. This open source license allows you to use protozoa as a true open source resource. You just have to make sure to give us attribution by keeping this file where it is, and not touching any of the developer meta tags in the html.

The (hidden) .htaccess.txt File

We use this file to force our web server to have *.SVG support. I assume that you do as well, but if you don’t then just delete this file. If you need to force Apache directives then add them to this file. NOTE: this file is hidden because it is named with a preceded period dot. Treat it like any other hidden file.

Browser Support

We used CSS3 components in this learning object.In other words—we support most modern browsers except IE8 and Opera Mini. That does not mean that this will not work with those two browsers just that you may not have the same experience as others while viewing content, and sometimes that can feel intrusive.

Accessibility

We have done our best to build in as many features as we could. We’ve tested these features across browsers and using standards based tools like WAVE. This template is solid, but that is never the whole story is it.

To put it bluntly, you can break accessibility by inserting non accessible content, or by not complying with best practice while inserting your content.

Semantics

The easiest thing you can do to remain accessible is by using semantic elements. Simply put, semantic elements describe their contents. This is as simple as using the paragraph tag when surrounding an paragraph of text. Use a heading tag when identifying a header or sub-header. So you need a bulleted list…use the unordered list tag. Need numbers on your list, then use the ordered list tag. It seems simple but people get caught up using the DIV tag to define everything.

It’s not like the DIV tag is never used, it is just used for structural elements. Like our column sets. For example. Add a DIV class=”row”, add two more divs inside that to define your columns. Then insert headers, paragraphs, or lists inside of those structures.

With the evolution of HTML to version 5 each module is an article. You will see that we already have this taken care of for you, but you are in charge of everything else. Like sections. The rule of thumb for the section element is to use it every time you add a different header level. Think of it as sectioning an outline.

Graphics

Alt tags are a subject that most people consider cut and dry. That cannot be further from the truth, and is best described by the Experts at WebAIM.

Tables

Tables are universally horrible on any mobile device you try to view them on. Currently there is no solution to this dilemma. We suggest that you break any tables out of your HTML content, and put them into a text based PDF that opens into a different tab or window. Those are not only search-able & printable, but also accessible.

3 The CSS

These are the styles you can use to layout a page very quickly, but there are a few things you should know first.

Rows

Rows are used to create new sections that span the whole page. It is good practice to insert at least one sub-header into a row along with a column set or six. You can see this in action in any of the markup examples located below.

Column Sets

Column set are meant to be easy and save you time so don't over think them. As you will see in the examples below—each comes as a set. Each set has been designed to be used with it's sibling, but you can break them apart and use them singly within other structures if you have the courage to come up with new patterns.

NOTE: We suggest that you use an html <section> element to make up those rows, but that's really up to & depends on the way you structure your content. Feel free to use a <div> as well. We use <section> because it implies a change of subject even if there is no sub-header to break up that content. By using the section element we still semantically imply the change of subject with the added benefit of keeping relevant content together.

4 A Word Regarding Customization

It's all in the SCSS

Protozoa has powerful customization features. You have the ability to change things like the colors, font families and weights, and card values. To access these features, at the very least, you will need to be familiar with compiling SASS. SASS is one of many CSS preprocessors out there that allow you to add logic and variable to CSS style sheets.

With that in mind you will find all the styles you need right at the top of the styles.scss file defined as SASS variables.

NOTE: You can find out all you need to know about SASS at the SASS homepage and in the SASS Basics Guide .

Colors

These colors are defined as variables right at the top of the style.scss file. They are defined as follows.

To change the colors just change the value between the colon and semi-colon on each line. Each color is applied to several different elements throughout protozoa. Some of those elements are visible, and some are hidden.

  • $primaryColor: #2F343B;

  • $lightPrimaryColor: #B8CCC7;

  • $darkPrimaryColor: #212423;

  • $accentColor: #877551;

  • $shadowColor: #727272;

  • $white: #F7FFFC;

  • $black: #090A0A

The visible changes are listed here so you can gain a better understanding of what you are changing while you are in the file, and so you don’t have to necessarily reload the page a thousand times.

  • Primary Color: Main Module Header banner background.

  • Light Primary Color: Main body background color

  • Dark Primary Color: Slide Header color, and Menu backgrounds.

  • Accent Color: Next/Previous Button background.

  • Shadow Color: all the drop shadows.

  • White: slide content area background color.\

  • Black: Slide Header color, Slide Title Color, main text color.

Why $black isn't black(#000000) and $white isn't white(#FFFFFF).

For the purposes of reading a large amount of text on screen, it’s easier on the eyes if the black is not true black, and the white is just off white.

How to choose the colors for your color palette.

The first thing we do is choose a primary color. All the other colors are chosen using color formulas based on that color. You can do this also using Adobe’s convenient color tool. Color.adobe.com. This tool allows you to choose a primary color then give you other colors to choose based on that color with a formula applied.

  • Start by choosing a primary color.

  • To choose an accent color select complementary from the color rule menu then pick one.

  • To choose a light and dark primary color select shades from the color rule menu then pick one.

  • For the Shadow color I choose a neutral gray every time.

  • For White: While still in the Shades color rule menu; pull your primary color directly towards the center of the color circle, but not perfectly in the middle. Just off perfect white, and hinting towards your color.

  • For Black: Start with your white value and grab the brightness slider and pull it till it is just above true black. So it is not completely black, but you can still see that it has a hint of your color. This will give you an almost black color that matches the rest of the palette.

5 The Javascript

Adding Your Own

If you would like, or need, to add your own behaviors via Javascript then please feel free to add them to the custom.js file. It will save you a bit of heartache, and debugging time if you just stay in that custom.js file Trust me. :)

Disclaimer

The main Javascript file is working hard in the background so you don't have to. It's doing things like filling up the Table of Contents based on the content you put in the slides, and applying ARIA roles for accessibility so you don't have to. This magic all happens in the main.js file. You can take a look, but it would not be a good idea to change anything in that file.
